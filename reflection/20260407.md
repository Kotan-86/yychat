## 3. SBIフィードバック（他者評価）

### SBI #1: 理解の深掘りと実装品質
- **Situation**: AVFoundation 導入初期で、セッション管理と音声合成の責務分離を設計していた。
- **Behavior**: `sharedInstance()` の意味、`setCategory` と `setActive` の順序、`speak` がキュー追加である点まで、疑問を細かく言語化して確認した。
- **Impact**: API 誤用のリスクが下がり、`Audio` と `Speech` を分離した実装方針が明確になった。以後の機能追加（入力連動、自動送信）の土台が強くなった。

### SBI #2: 不具合への対処
- **Situation**: `Unexpectedly found nil while implicitly unwrapping an Optional value` が発生し、画面遷移初期でクラッシュしていた。
- **Behavior**: `assert` と接続確認の手順で原因を絞り、`IBOutlet` 未接続を特定して再接続した。
- **Impact**: 原因特定が早まり、同種障害の診断パターンが獲得できた。実装継続の中断時間を最小化できた。

### SBI #3: チーム運用上の基礎整備
- **Situation**: 実装と並行して Git 運用（`.gitignore`、追跡対象整理）を進める必要があった。
- **Behavior**: `xcuserdata` など追跡不要ファイルの扱いを確認し、コミット前の状態を段階的に点検した。
- **Impact**: リポジトリのノイズが減り、今後の差分レビューや共同開発の摩擦を抑える基盤が整った。
